1
Install Node.js (one time only)
The engine that runs React on your computer

Think of Node.js like the engine of a car — React needs it to run. You only install it once.

  1. Go to nodejs.org
  2. Click the big green LTS button (Long Term Stable — the safe choice)
  3. Download and install it like any normal app
  4. Check it worked — open your Terminal (Mac) or Command Prompt (Windows) and type:
node --version

If it shows something like v20.x.x — you're good!

💡 On Windows, search "Command Prompt" in the Start menu. On Mac, search "Terminal" in Spotlight (Cmd+Space).
2
Create a React project and add your JSX
Wrap your CRM code in a proper React app

Your .jsx file is like a cake. This step puts it in a box so it can be shipped (deployed).

  1. Open Terminal / Command Prompt
  2. Navigate to your Desktop (or wherever you want to work):
cd Desktop
  1. Create a new React project:
npx create-react-app my-crm
  1. Enter the project folder:
cd my-crm
  1. Open the folder in your file explorer. Find the file src/App.js
  2. Delete everything inside it and paste your CRM's JSX code in there
  3. Save the file. Then test it works locally:
npm start

A browser window should pop open at http://localhost:3000 showing your CRM. 🎉

💡 Press Ctrl+C in the terminal to stop the local server when you're done testing.
3
Put your code on GitHub
GitHub is like Google Drive but for code

Vercel needs your code on GitHub to deploy it. Think of GitHub as the middleman.

  1. Go to github.com and create a free account
  2. Click the green New button → Name it "my-crm" → Click "Create repository"
  3. Download GitHub Desktop (free app — much easier than terminal)
  4. Open GitHub Desktop → Sign in → Click "Add existing repository"
  5. Point it to your my-crm folder on your Desktop
  6. Click Publish repository → make sure "Keep this code private" is checked if you want → Click Publish
💡 GitHub Desktop is a free visual app so you don't need to type complicated git commands. Highly recommended for beginners!
4
Deploy your website FREE on Vercel
Go from code on GitHub → live website in 2 minutes
Free foreverNo credit card needed
  1. Go to vercel.com
  2. Click Sign Up → Choose Continue with GitHub
  3. Click New Project
  4. Find your "my-crm" repo in the list → Click Import
  5. Vercel will auto-detect it's a React app — don't change any settings
  6. Click the big Deploy button
  7. Wait about 30-60 seconds...
  8. 🎉 Your site is LIVE at something like my-crm.vercel.app
💡 Every time you update your code and push it to GitHub (via GitHub Desktop), Vercel automatically rebuilds and updates your live site within 30 seconds!
5
Add a FREE database with Firebase
So your CRM actually saves and loads data

Without a database, your CRM data disappears every time you refresh the page. Firebase fixes that — it's free and made by Google.

Choose your database:

🔥 Firebase (recommended)

Google's free database. No backend needed — works directly in React. Best for beginners. Free up to 1GB storage + 50k reads/day.

EasiestReal-time
⚡ Supabase (alternative)

Open-source alternative to Firebase. Uses SQL (spreadsheet-style tables). 500MB free storage. Great if your CRM has complex data relationships.

Free tierSQL

Setting up Firebase (step by step):

  1. Go to console.firebase.google.com
  2. Sign in with Google → Click Create a project → Name it "my-crm"
  3. Disable Google Analytics (not needed) → Click Create project
  4. In the left sidebar click Firestore Database → Create database
  5. Choose Start in test mode (allows open access for testing)
  6. Pick any location close to you → Click Enable
  7. Go back to Project Overview → Click the </> (web) icon to register your app
  8. Give it a name like "my-crm-web" → Click Register app
  9. Copy the config code that appears — it looks like this:
const firebaseConfig = {
  apiKey: "AIza...",
  authDomain: "my-crm.firebaseapp.com",
  projectId: "my-crm",
  storageBucket: "my-crm.appspot.com",
  messagingSenderId: "123456",
  appId: "1:123456:web:abcdef"
};
  1. In your React project, install Firebase:
npm install firebase
  1. Create a new file src/firebase.js and paste this:
import { initializeApp } from "firebase/app";
import { getFirestore } from "firebase/firestore";

const firebaseConfig = {
  // paste YOUR config values here
};

const app = initializeApp(firebaseConfig);
export const db = getFirestore(app);
⚠️ Test mode is open to everyone for 30 days. Before sharing with real users, go to Firestore → Rules and add security rules so only you can read/write data.
6
Read & write CRM data to Firebase
The actual code to save contacts, leads, notes etc.

Inside your CRM JSX file, import Firebase and use it like this:

import { db } from './firebase';
import { collection, addDoc, getDocs } from 'firebase/firestore';

// SAVE a new contact to cloud:
const saveContact = async (contact) => {
  await addDoc(collection(db, "contacts"), contact);
};

// LOAD all contacts from cloud:
const loadContacts = async () => {
  const snapshot = await getDocs(collection(db, "contacts"));
  return snapshot.docs.map(doc => ({ id: doc.id, ...doc.data() }));
};
💡 Each "collection" in Firestore is like a spreadsheet tab. You can have "contacts", "deals", "notes" — each is its own collection.
7
Push updates → site auto-updates
Your workflow from now on

Once everything is set up, your daily workflow to update the site is:

  1. Make changes to your code in src/App.jsx
  2. Open GitHub Desktop
  3. You'll see your changes listed — write a short message like "fixed contact form"
  4. Click Commit to main
  5. Click Push origin
  6. Go to your Vercel dashboard — it'll show "Building..." for ~30 seconds
  7. Done! Your live site is updated ✅
💡 You can also do this from the terminal with:
git add . && git commit -m "your message" && git push
